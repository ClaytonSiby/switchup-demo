⚡ What is this app?
▶

This is a self-healing web scraper. It visits a website, collects data, and automatically detects when something breaks — for example, when a website redesign causes a field like "price" to go missing.

When a breakage is detected, it asks an AI to analyse the page and suggest a fix. The fix is saved as a patch file for a developer to review and apply. Nothing is changed automatically.

▶ Running the scraper
▶
1
Select a provider from the dropdown at the top — this is the website you want to scrape.
2
Toggle Demo on if you want to see the full healing flow. Demo mode uses a deliberately broken selector so you can watch the AI fix it.
3
Click Run. The pipeline log on the left will show live progress.
4
Results appear in the Latest Run tab on the right. If a fix was found, a diff preview will also appear at the bottom of the log.
Tip: Leave Demo mode off for a normal scrape — the healer will only fire if the live website actually has a broken selector.
🔄 What each pipeline step means
▶
Scrape — The app opens the website in a headless browser and collects the raw data.
Validate — Each item is checked to make sure all required fields have values.
Detect — Any fields that came back empty or missing are flagged as failures.
Heal — The AI receives a snapshot of the page HTML and suggests a corrected CSS selector.
Patch — The suggested fix is tested against real data, then saved as a patch file ready to apply.
📋 Understanding the results
▶

⚠ Something went wrong — A field could not be scraped. The card shows which field broke and the error reason.

✦ AI found a fix — The AI proposed a new CSS selector. The confidence bar shows how certain it is. Read the diagnosis for a plain-English explanation of what changed on the page.

✓ Fix was tested — The proposed selector was tried against the real page. "20 of 20 items passed" means it works perfectly. Fewer than 100% means you should review carefully.

✔ Fix is ready to apply — A patch file has been saved. Share the git apply command with your developer, or use the copy button in the diff preview at the bottom of the screen.

⚙ Adding a new website
▶
1
Click the ⚙ button in the top right to open the provider settings.
2
Click + Add Provider and fill in a name, the page URL, and the CSS selector for the item container (e.g. each product card).
3
Add a row for each field you want to collect (e.g. title, price). Use the Test button to check each selector works before saving.
4
Optionally add a Demo Break — a deliberately wrong selector — so you can demo the healing flow for this provider.
Not sure what a CSS selector is? Ask your developer to help fill in the selectors. Once saved, you can run the scraper yourself without any further help.
✔ Applying a fix
▶

When a fix is ready, a diff preview appears at the bottom of the pipeline log. The app never applies fixes automatically — a human must review and approve every change.

To apply a fix, copy the git apply command from the diff panel and run it in a terminal from the project folder — or forward it to your developer.

After applying, run the scraper again in Live mode (Demo off) to confirm the fix works on the real site.

Why isn't it automatic? Auto-applying AI-generated changes to a live scraper is risky. The confidence score and sandbox test reduce the chance of a bad fix, but a human review step is always the safest approach.
